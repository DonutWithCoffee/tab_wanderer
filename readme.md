# tab_wanderer

Chrome extension для автоматического отслеживания заказов в админке amperkot.ru.

---

## О проекте

tab_wanderer — это инструмент для мониторинга заказов в режиме реального времени.

Расширение автоматически:
- отслеживает изменения заказов
- фиксирует новые заказы
- уведомляет пользователя о событиях
- позволяет быстро открыть заказ по клику из уведомления

Основная задача — дать менеджеру прозрачный контроль над происходящим в системе без необходимости постоянно обновлять страницу.

---

## Текущий статус

Проект находится в стадии **stable core / engine**.

Это означает:
- базовая архитектура уже стабильна
- ключевые механики работают предсказуемо
- дальнейшее развитие идёт через расширение логики, а не переписывание ядра

Текущая версия: **0.9.3-dev**

---

## Как работает расширение

1. Создаётся специальная скрытая вкладка (worker)
2. Она регулярно загружает список заказов
3. Content script собирает данные со страницы
4. Система сравнивает текущее состояние заказов с предыдущим
5. Если обнаружены изменения:
   - новые заказы всегда фиксируются
   - для остальных применяется система правил
6. Если событие считается значимым — показывается уведомление
7. При клике на уведомление открывается соответствующий заказ

---

## Какие данные отслеживаются

Для каждого заказа фиксируются:

- номер заказа
- статус
- доставка
- оплата

Дополнительно (служебно, без отображения в UI):

- дата
- контрагент

Эти данные используются для определения изменений и работы правил.

---

## Логика уведомлений

Все уведомления проходят через систему правил.

Правила позволяют:
- игнорировать определённые типы заказов
- фильтровать события по статусу, доставке и оплате
- комбинировать условия

Примеры:
- игнорировать самовывоз
- игнорировать отменённые заказы
- игнорировать безналичные оплаты

При этом:
- **новые заказы всегда считаются значимым событием**
- изменения существующих заказов обрабатываются через правила

## Архитектура

Проект разделён на три уровня:

### Core (ядро)

Стабильная часть системы:

- управление worker-вкладкой
- восстановление после перезапуска браузера
- хранение состояния заказов
- определение изменений через hash
- watchdog (контроль работоспособности)

Ядро меняется только при необходимости.

---

### Rules Layer (правила)

Гибкий слой логики:

- определяет, нужно ли уведомление
- содержит правила фильтрации
- легко расширяется без изменения ядра

---

### Feature Layer (функции)

Пользовательские возможности:

- настройки
- конфигурация
- UI

Этот слой будет расширяться дальше.

---

## Текущий приоритет развития

Проект развивается по этапам:

1. Доработка правил уведомлений
2. Введение модели событий (тип изменения заказа)
3. Добавление пользовательской конфигурации
4. Улучшение наблюдаемости (история и диагностика)

---

## Требования к версии 1.0

Версия 1.0 считается готовой, когда:

- worker стабильно работает без ручного вмешательства
- нет дублирования вкладок
- корректно работает восстановление после перезапуска
- изменения заказов определяются точно
- нет ложных уведомлений
- правила легко расширяются
- пользователь может управлять уведомлениями
- интерфейс минимален и понятен
- поведение расширения не требует постоянного контроля
